# GitHub PR

PR品質を繰り返しチェックし、マージ判断を行う。

## 開発フローとスキルの役割

1. 実装・push・PR作成
2. **セルフレビュー** → readyにしてレビュー依頼
3. レビュワーとの対話・ブラッシュアップ
4. **セルフレビュー**（都度実行）
5. レビュワーからのApprove
6. **セルフレビュー**（最終確認）
7. **マージ** → リリース・動作確認

太字の箇所でこのスキルを使用。
